# Security Policy

## Supported Scope

Security reports are welcome for:

- the local web UI
- SMTP and IMAP credential handling
- preview sanitization and attachment handling
- session handling and remote-access protections
- any issue that could expose private message content or credentials

## Reporting a Vulnerability

Do not open public GitHub issues for security problems.

Use GitHub's private vulnerability reporting feature for the repository if it is enabled. If it is not enabled, contact the repository owner directly through the contact information on the repository profile and include:

- a short description of the issue
- impact and attack conditions
- reproduction steps
- affected files or endpoints
- any suggested mitigation

## Disclosure Expectations

- Please give the maintainer reasonable time to investigate and ship a fix before public disclosure.
- If the report involves mailbox credentials or message content, rotate affected secrets immediately after testing.
